# AppManager & UpdateServer 技术栈说明文档

> 文档生成日期：2026-03-04
> 覆盖范围：客户端 AppManager + 服务端 UpdateServer 完整技术栈

---

## 1. 系统概述

本系统由两个独立的 Qt C++ 应用共同组成一套完整的**多应用集中管理与在线升级平台**：

| 组件 | 项目名 | 角色 |
|------|--------|------|
| 客户端 | **AppManager** | 应用启动台 + 在线升级客户端 |
| 服务端 | **UpdateServer** | HTTP 升级包分发服务器 + 管理面板 |

**通信方式**：客户端通过标准 HTTP GET 请求从服务端拉取版本元数据和升级包文件。

### 3.3 升级模式详解

#### 模式一：EXE 替换升级

```
下载 EXE → SHA256 校验 → 写入 .new 文件 → 备份至 .bak → 删除旧 EXE → 重命名 .new → 完成
```

#### 模式二：ZIP 解压升级

支持三种子模式：

| 子模式 | 对应字段 | 行为 |
|--------|----------|------|
| **single** | `installMode = "single"` | 解压到指定目录，覆盖非 EXE 文件，递归替换 EXE |
| **allApps** | `installMode = "allApps"` | 解压到 appsRoot，按 `apps.json` 中所有应用的 EXE 路径逐一替换 |
| **清单替换** | `zip_replace_manifest.json` | 按清单文件列表精确替换指定文件 |

ZIP 解压升级流程：
```
下载 ZIP → SHA256 校验 → PowerShell 解压到临时目录 →
步骤1: syncMissingFilesAndDirs（补齐缺失目录和文件） →
步骤2: 清单模式检查或跳过 →
步骤3: 清单替换/默认覆盖 + EXE 递归替换 → 完成
```

### 3.4 依赖修复机制

```
服务端配置 requiredFiles 列表 →
客户端升级前检查文件是否完整 →
    完整 → 正常升级
    不完整 → 下载 fullPackageUrl 完整包 →
        PowerShell 解压 → 递归复制所有文件到目标 →
        二次校验 → 通过后继续升级
```

### 4.2 核心类与职责

#### 4.2.1 HttpServer（HTTP 服务器）

**架构**：基于 `QTcpServer` 的异步事件驱动 HTTP 服务器。

**路由表**：

| 路由 | 功能 | 响应类型 |
|------|------|----------|
| `GET /updates/{appId}.json` | 返回静态升级元数据 | `application/json` |
| `GET /download/{fileName}` | 返回升级包二进制文件 | `application/octet-stream` / `application/zip` / `application/x-msdownload` |
| `GET /status` | 服务器运行状态监控 | `application/json` |
| `GET /` | 健康检查 | `application/json` |

**安全特性**：

| 特性 | 实现 |
|------|------|
| **路径穿越防护** | `isPathSafe()`：拒绝含 `..`、`/`、`\` 的文件名 + `QDir::cleanPath` 验证解析路径在 baseDir 内 |
| **管理界面隔离** | HTTP 仅暴露 GET 只读接口，所有 CRUD 操作通过本地 Qt GUI 完成 |
| **请求大小限制** | `MAX_REQUEST_SIZE = 8 KB`，超过返回 413 |
| **连接数限制** | 默认 200 上限，超过拒绝新连接并记录日志 |
| **空闲超时** | 每连接 `QTimer`（30 秒），空闲自动断开 |
| **连接数告警** | 达到 80% 容量时通过 `connectionWarning` 信号通知 |

**异步文件传输**（重点架构改造）：

```
旧方案（同步阻塞）:
    while(remaining > 0) { socket->write(chunk); socket->waitForBytesWritten(); }

新方案（异步事件驱动）:
    FileTransferState { QFile*, remaining }
    sendFile() → 写入 HTTP 头 → 创建 FileTransferState → sendNextChunk()
    bytesWritten 信号 → sendNextChunk()（缓冲区 < 256KB 时继续写入）
    remaining == 0 → 关闭文件 → disconnectFromHost
```

关键参数：
- 分块大小：64 KB（`FileTransferState::CHUNK_SIZE`）
- 写缓冲区高水位：256 KB（`WRITE_BUFFER_HIGH`）

**HTTP Range 支持**：解析 `Range: bytes=start-end` 请求头，返回 `206 Partial Content` + `Content-Range` 响应头。

**连接生命周期管理**：

```
newConnection →
    检查连接数上限 → 插入 m_connections →
    创建 QTimer（空闲超时）→ 连接 readyRead / disconnected / bytesWritten 信号

readyRead → 重置空闲计时器 → 检查请求大小 → handleRequest()

disconnected → cleanupConnection()（清理 FileTransferState + QTimer + 从集合移除）→ deleteLater()

超时 → cleanupConnection() → socket->abort() → deleteLater()
```

## 10. 安全机制总结

| 层面 | 机制 | 实现 |
|------|------|------|
| **传输安全** | 目前使用 HTTP 明文（内网场景），公网部署需升级 HTTPS 或前置 Nginx |
| **路径安全** | `isPathSafe()` 双重校验：字符检查 + 绝对路径归属验证 |
| **管理隔离** | CRUD 操作仅限本地 GUI，HTTP 层无写入接口 |
| **文件完整性** | SHA256 校验（下载后对比服务端声明值） |
| **进程安全** | 升级前检测目标进程，关闭前确认，升级后自动重启 |
| **连接防护** | 200 连接上限 + 30s 空闲超时 + 80% 预警 |
| **请求防护** | 8 KB 请求头大小限制 |
| **数据备份** | EXE 升级前创建 `.bak` 备份，失败自动回滚 |
| **日志审计** | 全量请求日志 + 文件持久化 + 10 MB 自动轮转 |

## 11. 性能与并发特性

| 特性 | 说明 |
|------|------|
| **事件驱动** | `QTcpServer` + 信号槽异步架构，单线程无阻塞 |
| **异步文件传输** | `bytesWritten` 信号驱动分块发送，避免阻塞事件循环 |
| **流量控制** | 写缓冲区高水位 256 KB，低于阈值时继续写入 |
| **连接管理** | `QSet` 跟踪活跃连接，`QMap` 管理传输状态和超时定时器 |
| **断点续传** | 客户端支持 HTTP Range 请求 + 最多 5 次自动重试 |
| **静态化** | 版本元数据预生成为静态 JSON 文件，避免运行时动态构建 |
