# UpdateServer 服务端适配 AppManager 自更新指南

## 概述

本指南说明如何在 UpdateServer 中配置和部署 AppManager 的自更新功能。

## 架构

```
客户端（AppManager）
       |
       v
  GET /updates/AppManager.json
       |
       v
  UpdateServer HTTP 接口
       |
       v
  server_apps.json 配置
       |
       v
  packages/AppManager/{安装程序}
```

## 步骤 1：添加 AppManager 条目到 server_apps.json

编辑 `server_apps.json`，添加如下条目：

```json
{
    "appId": "AppManager",
    "appName": "AppManager",
    "latestVersion": "1.0.0",
    "packageFileName": "AppManagerSetup_1.0.0.exe",
    "sha256": "",
    "packageType": "exe",
    "subDir": "",
    "zipReplaceExeRecursively": false,
    "requiredFiles": [],
    "fullPackageFileName": ""
}
```

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| `appId` | 应用唯一标识，客户端用此查询更新 | `AppManager` |
| `appName` | 应用显示名称 | `AppManager` |
| `latestVersion` | 最新版本号（语义化版本） | `1.0.0` 或 `2.0.1` |
| `packageFileName` | 安装程序文件名 | `AppManagerSetup_1.0.0.exe` |
| `sha256` | 文件 SHA256 哈希值（十六进制小写） | 自动计算，可空 |
| `packageType` | 升级包类型 | `exe` 或 `zip` |
| `subDir` | 客户端安装子目录（相对 appsRoot） | 为空表示跟目录 |
| `zipReplaceExeRecursively` | ZIP 升级时是否递归替换 exe | `false`（exe 无需） |
| `requiredFiles` | 依赖文件列表 | `[]`（通常为空） |
| `fullPackageFileName` | 完整包文件名（可选） | `""` |

## 步骤 2：准备升级包文件

1. **编译安装程序**
   ```bash
   cd d:\WorkSpace\QtProjects\AppManager\installer
   .\build_installer.ps1 -Version 1.0.0
   ```
   输出：`AppManager\installer_output\AppManagerSetup.exe`

2. **重命名并放置到服务端**
   ```bash
   # 源文件
   AppManager/installer_output/AppManagerSetup.exe
   
   # 目标路径（以版本号命名）
   UpdateServer/packages/AppManager/AppManagerSetup_1.0.0.exe
   ```

3. **创建目录**
   ```bash
   mkdir UpdateServer/packages/AppManager
   copy AppManager/installer_output/AppManagerSetup.exe UpdateServer/packages/AppManager/AppManagerSetup_1.0.0.exe
   ```

## 步骤 3：启动服务端并生成静态元数据

### GUI 模式（推荐）

1. 启动 `UpdateServer.exe`
2. 在表格中应该看到 `AppManager` 行
3. 在 MainWindow 中点击"刷新"或"重新生成元数据"按钮

### 无头模式

```bash
UpdateServer.exe --headless --port 6655
```

服务端会自动：
1. 加载 `server_apps.json`
2. 读取 `packages/AppManager/AppManagerSetup_1.0.0.exe`
3. 计算 SHA256
4. 生成 `updates/AppManager.json`

### 验证生成的元数据

检查 `UpdateServer/updates/AppManager.json`：

```json
{
    "latestVersion": "1.0.0",
    "downloadUrl": "http://your-ip:6655/download/AppManagerSetup_1.0.0.exe",
    "packageType": "exe",
    "sha256": "abc123...",
    "subDir": "",
    "requiredFiles": [],
    "fullPackageFileName": ""
}
```

## 步骤 4：配置客户端

编辑客户端的 `apps.json`，确保 `serverBaseUrl` 指向服务端：

```json
{
    "serverBaseUrl": "http://your-server-ip:6655",
    "appsRoot": ".",
    "apps": [...]
}
```

## 步骤 5：测试自更新流程

1. 启动 AppManager 客户端
2. 菜单 → 帮助 → 检查 AppManager 更新
3. 验证以下步骤：
   - ✅ 连接服务器成功
   - ✅ 读取 `/updates/AppManager.json` 成功
   - ✅ 版本比较正确
   - ✅ 下载安装程序成功
   - ✅ 安装程序启动

## 常见问题

### Q: SHA256 字段如何获取？

**A：**自动计算。在 UpdateServer UI 中操作时，系统会自动：
1. 读取 `packages/AppManager/AppManagerSetup_*.exe`
2. 计算 SHA256
3. 写入 `server_apps.json`

或手动计算：
```powershell
(Get-FileHash "UpdateServer\packages\AppManager\AppManagerSetup_1.0.0.exe" -Algorithm SHA256).Hash
```

### Q: 如何更新到新版本？

**A：**遵循以下步骤：

1. **编译新版本**
   ```bash
   cd AppManager
   git pull  # 获取最新代码
   qmake && mingw32-make
   ```

2. **生成新安装程序**
   ```bash
   cd installer
   .\build_installer.ps1 -Version 2.0.0
   ```

3. **放置到服务端**
   ```bash
   copy AppManager\installer_output\AppManagerSetup.exe `
        UpdateServer\packages\AppManager\AppManagerSetup_2.0.0.exe
   ```

4. **更新 server_apps.json**
   ```json
   {
       "appId": "AppManager",
       "latestVersion": "2.0.0",
       "packageFileName": "AppManagerSetup_2.0.0.exe",
       ...
   }
   ```

5. **重启服务端**（或点击"刷新"按钮）
   
   服务端会自动生成新的 `updates/AppManager.json`

6. **测试**
   
   客户端下次检查更新时会发现新版本

### Q: 多个版本可以共存吗？

**A：**可以。`packages/AppManager/` 目录中可以存放多个版本：

```
packages/AppManager/
├── AppManagerSetup_1.0.0.exe
├── AppManagerSetup_1.0.1.exe
└── AppManagerSetup_2.0.0.exe
```

但 `server_apps.json` 中只能指定一个 `latestVersion`。旧版本可用于回滚或灾备。

### Q: 如何实现灰度发布？

**A：**服务端需要扩展 `/updates/AppManager.json` 接口。当前实现是静态文件，可增强为：

```python
# 伪代码示例
GET /updates/AppManager.json?clientId={uuid}
  if clientId in canary_group:
      return version 2.0.0
  else:
      return version 1.0.0
```

修改方案：升级 UpdateServer 的 HttpServer 以支持动态查询。

### Q: 客户端提示"检查更新失败"

**A：**排查项：

1. **网络连接**
   ```bash
   ping your-server-ip
   ```

2. **URL 可访问**
   ```bash
   curl http://your-server-ip:6655/updates/AppManager.json
   ```

3. **JSON 格式**
   ```bash
   # 验证 JSON 有效性
   ```

4. **服务端日志**
   ```bash
   # 查看 UpdateServer 日志（GUI：日志窗口）
   # 或 server.log 文件
   ```

### Q: 如何实现强制更新？

**A：**在 `server_apps.json` 中添加扩展字段（需修改客户端代码）：

```json
{
    "appId": "AppManager",
    "latestVersion": "2.0.0",
    "forceUpdate": true,
    "minSupportedVersion": "1.0.0"
}
```

客户端逻辑：
```cpp
// 如果版本 < minSupportedVersion，禁用"稍后提醒"，强制升级
```

## 部署架构参考

### 单机服务端部署

```
┌─ UpdateServer
│  ├─ UpdateServer.exe
│  ├─ server_apps.json       ← 配置文件
│  ├─ packages/              ← 升级包目录
│  │  ├─ AppManager/
│  │  │  └─ AppManagerSetup_1.0.0.exe
│  │  ├─ cantest/
│  │  └─ BCTester/
│  ├─ updates/               ← 生成的静态元数据
│  │  ├─ AppManager.json     ← 由服务端自动生成
│  │  ├─ cantest.json
│  │  └─ bctester.json
│  └─ server.log
```

### 跨机器访问

客户端配置：
```json
{
    "serverBaseUrl": "http://192.168.1.100:6655"
}
```

### 云端部署

如果部署到云服务器，确保：
- 云主机的 6655 端口出站开放
- 防火墙规则允许 HTTP 访问
- 客户端配置正确的公网 IP 地址
